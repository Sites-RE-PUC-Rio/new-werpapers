Metamorfosis: Un Marco para el Análisis de Requisitos Funcionales

Isabel Díaz; Juan Sánchez; Oscar Pastor

Workshop on Requirements Engineering (WER)2005

Leer PDFVer ediciónGoogle Scholar

Resumen

Este artículo describe el Modelo de Realización de Metamorfosis. El propósito principal de este marco es proporcionar la base conceptual necesaria para el establecimiento de vínculos persistentes y controlables entre la especificación de los requisitos funcionales del sistema (expresado como un Modelo de Casos de Uso) y el producto de su análisis (representado a través de un Modelo de Objetos y un Modelo de Interacción). Metamorfosis permite la deducción automática de los modelos del análisis a partir del texto de los casos de uso. Con este objetivo, el Modelo de Casos de Uso ha sido enriquecido para hacer posible la representación de la información sintáctica y semántica de las acciones de los casos de uso. La utilización de roles para facilitar la comprensión de las representaciones y potenciar la genericidad, es otra característica de Metamorfosis.
